Данный ресурс предназначен для генерирования паролей, согласно заданной политике.
Кроме ряда предустановленных политик, можно также менять данные через запрос.

Краткое описание

Для получения пароля нужно посредством GET запроса к API ( {{ request.scheme }}://{{ request.host }}{{ api_url }} ) передать необходимые данные для генерирования пароля. Результат будет возвращен в видет JSON

Пароль сгенерирован 
                                                    {
                                                        "status": "success",
                                                        "password": "Fx1iEdyf",
                                                        "policy": "AD"
                                                    }
                                                     
Ошибка генерации 
                                                    {
                                                        "status": "error",
                                                        "password": "",
                                                        "policy": "AD",
                                                        "error": "Error message be here."
                                                    }
  1. status - cтатус операции (success, fail)
  2. password - сгенерированный пароль
  3. policy - политика генерации пароля
  4. error - сообщение об ошибке, если вернулся статус fail

Если в запросе не была указана политика генерации паролей, тогда используется политика по умолчанию (DEFAULT)
Количество символов в пароле - 12
Используемые символы: ABCDEFGHJKLMNPQRSTUVWXYZabcdefghijkmnopqrstuvwxyz23456789?#*)%

Использование API #к началу

Для генерирования паролей используются политики (набор правил, где указаны символы, которые можно использовать, а также длина пароля) Есть следующие готовые политики

  • AD
  • VPN
  • DB
  • PR
  • MAIL
  • DEFAULT

Для изменения количества символов в генерируемом пароле нужно добавить параметр chars_count к запросу с указанием нужного числа символов
Например, если в мы хотим сгенерировать пароль для политики AD, но с количеством символов 16, а не 8:
{{ api_url }}?policy=AD&chars_count=16

Получение пароля для AD - #к началу

Строка запроса: {{ api_url }}?policy=AD
Длина пароля: {{ policies["AD"]["CHARS_COUNT"] }} символов
Набор символов для генерации пароля: 

                                        {{ policies["AD"]["CHARS_ALLOWED"] }}
